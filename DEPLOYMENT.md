# Deployment Guide

## Prerequisites
- Docker Engine & Docker Compose
- PostgreSQL 15+ (If not using Docker)
- Redis 7+ (If not using Docker)
- Python 3.11+

## Steps to Deploy

1. **Clone & Environment Setup**
   Copy `.env.example` to `.env` and fill in your keys:
   ```bash
   cp .env.example .env
   # Edit GEMINI_API_KEY and TELEGRAM_BOT_TOKEN
   ```

2. **Database Migration**
   Initialize the PostgreSQL Schema using Alembic:
   ```bash
   cd db_migrations
   alembic upgrade head
   ```

3. **Boot Infrastructure**
   Start the Event Queue, n8n orchestrator, and Python Worker Pool:
   ```bash
   docker-compose up -d --build
   ```

4. **Verify Workers**
   Check the Python Worker logs to ensure it registered all MVP Capabilities successfully:
   ```bash
   docker logs acb_python-worker_1
   ```
   You should see: `Registered capabilities: ['analytics', 'strategy_planning', 'quality_gate', ...]`

5. **Import n8n Workflows**
   - Access n8n at `http://localhost:5678` (user/pass: admin/admin)
   - Click "Import from File" and select `workflows/n8n/04_business_flow.json`.
   - Set the Cron node to trigger daily.

6. **Production Notes**
   - Ensure you are monitoring the `q_dead_letter` queue in PostgreSQL for failed tasks.
   - You can review costs generated by `CostEngine` directly in the `costs` database table.
